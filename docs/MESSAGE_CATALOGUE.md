# Message Catalogue

> Living reference of every WebSocket message in Starbridge.
> Generated by v0.06-audit (2026-02-24). Keep in sync with code changes.

## Architecture

All messages use the `Message` envelope: `{type, payload, tick?, timestamp}`.

**Client → Server routing:**
1. Client sends `{type: "prefix.action", payload: {...}}`
2. `main.py._HANDLERS` maps `type.split(".")[0]` → handler function
3. Handler validates payload via `validate_payload()` (checks `_PAYLOAD_SCHEMAS`)
4. Validated payload is queued as `(type, payload)` for game loop processing

**Special cases:**
- `captain.*`: Some handled directly (set_alert, system_override, reassign_crew, save_game), others forwarded to queue (authorize, add_log, undock)
- `lobby.*`: Handled by lobby module, no game loop queue
- `game.*`: Handled by `_handle_game_message` in main.py
