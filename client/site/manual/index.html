User Manual

STARBRIDGE BRIDGE CREW SIMULATOR — REFERENCE EDITION

1. Getting Started

Joining a game. Navigate to the lobby at starbridge/lobby. Enter your callsign (player name) and press ENTER or click CONFIRM. The lobby displays all available station roles and which players currently hold them.

Claiming a role. Click CLAIM next to any unclaimed station. You may hold only one role at a time; clicking a second station will prompt you to switch. Once claimed, a RELEASE button appears so you can free the seat for someone else. Some ship classes require a minimum crew count before the host can launch.

Selecting a ship class. The host chooses a ship class before launching. Different classes have different hull ratings, torpedo loadouts, and crew requirements. See the Ship Classes section for details.

Choosing difficulty. The host selects a difficulty preset from the lobby dropdown: Cadet, Officer (default), Commander, or Admiral. Difficulty affects enemy strength, resource availability, repair speed, scan times, event pacing, and more. Hover over each option for a summary. See the Difficulty Levels section for the full parameter table.

Player profiles. Your callsign is tracked across sessions. After each mission, your profile updates with games played, wins, and career achievements. The lobby footer shows your win count. Click LEADERBOARD to see crew rankings.

The briefing screen. After the host clicks LAUNCH, all players are directed to the briefing room. The mission name and objectives appear alongside a 30-second countdown. The Captain can override the countdown by clicking LAUNCH NOW. Use this time to coordinate with your crew.

Basic concepts.

  • Game tick: The server runs at 10 ticks per second. All position, damage, and mission logic updates happen every tick.
  • Power budget: Nine ship systems share a budget of 900 power units. The default allocation is 100 per system. Engineering can redistribute power freely, but the total cannot exceed 900.
  • System efficiency: A system's efficiency is proportional to its power allocation, modified by health and crew factor. A system at 50 power operates at roughly 50% efficiency.
  • Hull: Hull points represent overall ship integrity. Hull damage is permanent within a mission (no passive regeneration). When hull reaches zero, the mission fails.
  • Shields: Shields are split into four facings — fore, aft, port, and starboard — sharing a total capacity of 200 HP. Shield distribution shifts based on a focus point set from Engineering. Shields regenerate automatically when powered and absorb incoming fire before hull is damaged; each facing absorbs hits independently.
  • Alert levels: GREEN (routine), YELLOW (enemy detected), RED (combat). Alert level is set by the Captain and affects ambient sound across all stations.
  • Crew factor: Each deck has a crew whose health affects system efficiency. Dead crew reduce performance permanently; injured crew have partial effectiveness.

2. Station Guides

CAPTAIN

Role summary: You are the ship's commanding officer. Your station synthesises all tactical information into a single overview. You set the alert level, maintain the captain's log, and are the sole authoriser for nuclear torpedo launches. Other stations report to you — you coordinate, not execute.

Controls

  • Alert Level selector: Sets ship-wide alert to GREEN, YELLOW, or RED. This changes ambient audio and visual colours across all stations simultaneously.
  • Tactical map (canvas): North-up overview showing your ship, enemy contacts, friendly units, and hazards. Does not scroll — world radius is 80,000 units. The bottom-right corner shows weapons crewing status: WEAPONS: CREWED (green) when a player occupies Weapons, or WEAPONS: AUTO (50%) (amber) when the auto-fire targeting computer is active.
  • Science summary panel: Condensed view of current sensor contacts from the Science station.
  • Hull / Shield readout: Real-time hull percentage and four-bar shield display (fore, aft, port, starboard) showing current HP per facing.
  • Captain's log: Type entries; they are timestamped and included verbatim in the post-mission debrief.
  • Nuclear authorisation panel: Appears when Weapons requests nuclear launch. Approve or deny within the countdown window.

Keyboard shortcuts: F1 — help overlay.

Tips: Set YELLOW when a contact appears on sensors; upgrade to RED only when actively firing. Narrate key moments in the log — the debrief awards bonus points for detailed records. Watch the shield readout — if shields fail while hull is low, call Engineering immediately.

HELM

Role summary: You fly the ship. You control heading and throttle, navigate hazard fields, and execute course changes requested by the Captain. Smooth manoeuvring keeps the shields pointed at the threat and reduces the chance of drifting into hazards.

Controls

  • Heading dial: Click-drag to rotate the ship. The outer ring shows the target bearing; the inner indicator shows current heading.
  • Throttle slider: Sets engine thrust 0–100%. Actual velocity depends on engine power from Engineering.
  • Bearing input (numeric): Type a bearing (0–360°) and confirm to set a precise heading.
  • Minimap: Compact sector overview showing ship position, contacts, hazards, and any plotted route from Navigation.
  • Docking approach: To dock at a station, reduce throttle to 10% or below while within the station's docking range, then request clearance. Friendly stations grant clearance within ~2 seconds; neutral stations within ~4 seconds. The docking sequence takes 10 seconds — hold course until complete.
  • Route Calculation puzzle: Triggered by missions that require navigating through hazard grids. Solve it to unlock a safe path.

Keyboard shortcuts: F1 — help overlay.

Tips: High throttle plus tight turns bleeds speed — you overshoot. Cut throttle before a sharp course change then increase again on the new heading. Nebulae don't cause damage but obstruct sensors — ask Science if you're flying blind. When approaching a station to dock, cut throttle well in advance; you must be below 10% before clearance can be requested. Gravity wells cap your speed to 100 u/s regardless of engine power — don't fight it, plot around them.

ENGINEERING

Role summary: You manage power distribution and system repairs. Nine systems share a fixed budget of 900 power units. You decide in real time which systems get priority. You also designate which system the repair crew focuses on. Balancing offence, defence, and propulsion is the core skill.

Controls

  • Power sliders (×9): One slider per system — Engines, Shields, Sensors, Weapons, Life Support, Flight Deck, ECM Suite, Point Defence, and a ninth system determined by ship class. The running total must not exceed 900.
  • Repair focus selector: Designates which damaged system the engineering crew repairs. Only one system can be repaired at a time; repairing a healthy system wastes crew time.
  • System health indicators: Each system shows current health (0–100 HP). Damaged systems operate at reduced efficiency even with full power.
  • Shield focus canvas: Drag the focus point toward the threat direction. Shield energy redistributes across four facings (fore, aft, port, starboard) based on the focus point. Use LOCK to freeze the focus and CENTRE to reset to an even split.
  • Circuit Routing puzzle: Triggered by mission events. Requires routing power through a grid to reconnect severed conduits.

Keyboard shortcuts: F1 — help overlay.

Tips: The power budget is strict — if you give Shields 150, something else must give up 50. Life Support below 50 power starts reducing crew health over time. In heavy combat: raise Shields and Weapons, lower Flight Deck and ECM. After combat: raise Sensors and Life Support for recovery. Drag shield focus toward the enemy bearing — concentrating shields on one facing is far more effective than spreading them evenly when under directional fire.

WEAPONS

Role summary: You operate the ship's offensive systems — beam cannons and torpedo tubes. You select targets, manage beam frequency to exploit shield vulnerabilities, load torpedo tubes with different warhead types, and handle nuclear authorisation requests from the Captain.

Controls

  • Radar display (canvas): Your ship at centre; contacts around it. Each contact is colour-coded by classification: hostile (red), neutral (amber), unknown (white, pulsing), and friendly (green, cannot be targeted). Click a hostile or unknown contact to lock target. Science scans reveal classification.
  • Beam fire button: Fires beam cannons at the locked target. Subject to cooldown after each burst.
  • Beam frequency selector: Four frequencies — ALPHA, BETA, GAMMA, DELTA. Matching the target's shield frequency deals 1.5× damage; mismatching deals 0.5×. Ask Science to scan the target.
  • Torpedo tubes (2): Each tube must be loaded before firing. Load time: 3 seconds to swap warhead type. Eight warhead types:
    • Standard — 50 dmg, 3 s reload. General purpose.
    • Homing — 35 dmg, 4 s reload. Tracks the locked target in flight.
    • Ion — 10 dmg, 5 s reload. Drains enemy shields and stuns for 10 s.
    • Piercing — 40 dmg, 4 s reload. Ignores 75% of shield absorption.
    • Heavy — 100 dmg, 8 s reload. Slow velocity; maximum single-hit damage.
    • Proximity — 30 dmg, 4 s reload. AOE blast — damages all enemies within 2,000 units.
    • Nuclear — 200 dmg, 10 s reload. Devastating; requires Captain authorisation before firing.
    • Experimental — 60 dmg, 6 s reload. Unpredictable secondary effects; resupply only.
  • Point Defence: Automated system (9th ship system) that intercepts incoming enemy torpedoes. Intercept chance scales with Point Defence efficiency. Engineering can allocate more power to improve coverage.
  • Firing Solution puzzle: Triggered by precision strike missions. Solve to unlock a torpedo firing angle against a moving target.
  • Auto-fire targeting computer: When no player occupies the Weapons station, an automated targeting computer engages after a 3-second delay. Auto-fire targets the nearest scanned hostile enemy within beam range and arc. It fires at half the manual cadence (1 shot/s vs 0.5 s), with 75% accuracy and no beam frequency matching — resulting in roughly 50% of a crewed Weapons station's effective output. Auto-fire uses beams only (no torpedoes) and does not change the manual target lock. When a player returns to Weapons, auto-fire disengages immediately and full manual control resumes. The Captain's tactical map shows the current status.

Keyboard shortcuts: F1 — help overlay.

Tips: Always coordinate with Science before engaging — knowing the shield frequency is a 3× swing in effective DPS (1.5× match vs 0.5× mismatch). Use Ion torpedoes to stun fast scouts before they close to beam range. Use Proximity against clustered enemies; use Piercing against heavily shielded targets. Keep both tubes loaded at all times during combat — a loaded tube fires instantly. Nuclear requires Captain authorisation — request it before combat, not during. Auto-fire is a safety net, not a replacement — it cannot match beam frequencies, fire torpedoes, or prioritise high-value targets. Return to Weapons as soon as the crisis on your other station is resolved.

SCIENCE / SENSORS

Role summary: You operate the sensor array, classify contacts, and run active scans that provide intelligence to all other stations. Your scans reveal shield frequencies for Weapons, hazard routes for Helm, and biological data for Medical. You also triangulate mission signal sources.

Controls

  • Sensor map (canvas): Sector-scale view with range rings. All contacts, hazards, drones, and probes are drawn here.
  • Scan mode selector: Four modes affect what each scan reveals as it progresses: EM (energy sources — stations, transponders), GRAV (mass concentrations — asteroids, gravity wells, derelicts), BIO (life signs — creatures, outbreaks, inhabited stations), SUB (subspace phenomena — anomalies, relays, cloaked contacts).
  • Sector sweep: Scans the current sector over 30 seconds, revealing features progressively. Interrupted if an enemy closes to within 15,000 units — you can dismiss the interrupt or continue at risk.
  • Long-range scan: Multi-sector sweep over 90 seconds, revealing adjacent sectors on the Captain's strategic map (Scanned status).
  • Contact list: Click any contact to initiate an active entity scan. Scan results broadcast to Weapons (shield frequency) and Captain.
  • Bearing compass: Multiple signal bearings drawn as dashed lines from scan origin — triangulate the source with two or more scans from different ship positions. Two bearings are required to locate a signal source.
  • Frequency Matching puzzle: Triggered by mission signal events. Tune waveform components to match an incoming transmission.

Keyboard shortcuts: F1 — help overlay.

Tips: Run EM mode in combat to reveal shield frequencies. Switch to BIO near creatures or during medical emergencies — active outbreaks and creatures produce biological signatures. Use BIO to confirm all hull leeches are removed before docking at a derelict. Long-range scans help Helm plan routes through unexplored sectors. Flight Operations drones extend your sensor range — coordinate with Flight Ops to cover blind spots.

MEDICAL

Role summary: You manage individual crew members, not just deck-level statistics. Every crew member has a name, a deck assignment, and can sustain specific injuries to body regions. Injuries degrade over time if untreated — a moderate wound becomes serious, serious becomes critical, and critical injuries are fatal. Dead crew are permanent losses. Your job is triage: decide who to treat first, what treatment to use, and when to quarantine contagious crew.

Casualty List (left panel)

  • Lists every crew member who has at least one injury, sorted by urgency (critical first) by default.
  • Each card shows: name, deck, worst severity (colour-coded), and injury count.
  • Severity colours: ■ Critical (red pulse) · ■ Serious (amber) · ■ Moderate (yellow) · ■ Minor (dim).
  • Sort buttons along the top let you reorder by urgency, deck, name, or arrival time. Filter by severity level or deck assignment.

Body Diagram (right panel)

  • When you select a crew member, a wireframe body diagram highlights injured regions. Colour matches the injury's severity.
  • Regions: Head, Torso, Left Arm, Right Arm, Left Leg, Right Leg. Click a region to filter the injury list to that area.
  • Below the diagram, each injury is listed with its type, severity, body region, and a treatment progress bar (if being treated).

Treatment Types

  • First Aid — fast, low cost. Suitable for minor and moderate injuries. No puzzle required.
  • Stabilise — buys time by pausing degradation on a critical or serious injury. Use this when beds are full to prevent death while you treat others. No puzzle required.
  • Surgery — required for serious injuries. Triggers a triage puzzle; success heals the injury, failure leaves it unchanged.
  • Intensive Care — required for critical injuries. Triggers a harder triage puzzle. High supply cost but saves lives.
  • Quarantine — isolates a contagious crew member in a quarantine slot. Prevents disease spread but does not heal — you must still treat the underlying injury.

Triage Prioritisation

  • Critical injuries degrade fastest and kill the crew member if untreated. Always stabilise or treat critical cases first.
  • Serious injuries degrade to critical over time — treat before they escalate.
  • When beds are full, stabilise the worst cases, then discharge healed crew to free beds for new patients.
  • Contagious crew should be quarantined immediately to stop the spread, then treated when a bed opens.

Crew Factor

Every deck has a crew factor — the ratio of effective crew to total crew. Active crew count fully; injured crew count at half. When crew factor drops, that deck's systems lose efficiency. An Engineering deck at 50% crew factor means slower repairs and weaker power output. Keeping crew healthy directly affects every other station's performance.

Status Bar

  • Beds: occupied / total treatment beds.
  • Queue: patients waiting for a free bed.
  • Supplies: remaining medical supplies (consumed by treatments; replenished at stations).
  • Quarantine: occupied / total isolation slots.
  • Morgue: total crew deaths this mission.
  • Crew: overall ship crew factor percentage.

Keyboard shortcuts: ↑/↓ — navigate casualty list · A — admit · D — discharge · Q — quarantine · S — stabilise · T — treat · 0–5 — deck filter · F1 — help overlay.

Tips: Stabilise critical crew immediately — it costs nothing and stops the death timer. Treat critical before serious, serious before moderate. Discharge healed patients promptly to free beds. Quarantine contagious crew even if you cannot treat them yet — one infected crew member can cascade into a ship-wide outbreak. Watch the crew factor percentage: if Engineering drops below 70%, your ship's repair capability is severely impaired.

SECURITY

Role summary: You coordinate internal ship defence against boarding parties and manage ship security systems. You command marine teams to intercept intruders, control door locks and deck lockdowns, manage emergency bulkheads, set alert levels, arm crew, and quarantine rooms.

Ship Interior Canvas

  • 5-deck × 4-room grid showing all rooms, door status, marine teams (blue ▲), and visible boarding parties (red ✕).
  • Click rooms to select; click with a marine team selected to send them there.
  • Deck tabs filter view to a single deck or show all.

Marine Teams

  • SEND: Select a team, click a room to dispatch them. Teams travel room-by-room and engage boarders automatically.
  • PATROL: Assign a patrol route — team cycles through rooms continuously.
  • STATION: Order a team to hold position.
  • DISENGAGE: Break contact during combat.
  • ESCORT: Assign a team to escort an Engineering repair team through hazardous areas.

Ship Security Systems

  • Door Control (L/U): Lock or unlock individual room doors. Locked doors block movement.
  • Lockdown: Lock all doors on a deck or ship-wide. Lift lockdown per-deck or all at once.
  • Bulkheads: Seal emergency bulkheads between adjacent decks to prevent inter-deck movement. Unsealing takes 30 seconds.
  • Alert Levels: Set per-deck: NORMAL, CAUTION, COMBAT (halves casualties), EVACUATE (reduces crew factor to 10%).
  • Armoury: Arm crew on up to 2 decks with sidearms for increased firepower against boarders.
  • Quarantine (Q): Seal a room and mark it quarantined. Blocks pathfinding through the room.

Keyboard shortcuts: L — lock door, U — unlock door, Q — quarantine, Tab — cycle teams, F1 — help overlay.

Tips: Use bulkheads to isolate decks during boarding. Set COMBAT alert on threatened decks to halve casualties. Arm crew on decks with active boarders. Coordinate with Comms — intercepted boarder communications reveal their objectives.

COMMUNICATIONS

Role summary: You manage faction diplomacy and encrypted transmission decoding. You tune to a faction's frequency band, select an appropriate message type, hail them, and interpret their responses. Intelligence from your transmissions feeds directly into mission outcomes.

Controls

  • Frequency scanner (canvas): Visual spectrum display showing faction signal spikes. Brighter spikes indicate which band you're closest to.
  • Frequency slider: Tune from 0.000 to 1.000. Lock onto a faction band within ±0.05 tolerance to enable hailing.
    Faction bands: Imperial 0.15, Rebel 0.42, Alien 0.71, Emergency 0.90.
  • Hail type: Negotiate, Demand, or Bluff. The correct choice depends on the faction's disposition and mission context.
  • Transmission log: All incoming NPC responses appear here with timestamps. Scroll to review earlier messages.
  • Contact ID input: Specify a vessel or station ID when hailing (e.g. "meridian_ship", "talon_ship").
  • Transmission Decoding puzzle: Triggered when an encrypted signal is intercepted. Break the cipher by solving sum equations linking cipher symbols to numeric values.

Keyboard shortcuts: F1 — help overlay.

Tips: Wrong message type can worsen faction relations mid-mission. Listen to NPC responses before hailing again — responses arrive 2 seconds after your transmission. The Frequency Matching puzzle on Science may receive relay data from your decoded transmissions; coordinate with Science when decoding.

FLIGHT OPERATIONS

Role summary: You command the ship's flight deck: launching, recovering, and directing drones on tactical missions. Five drone types (scout, combat, rescue, survey, ECM) cover sensor extension, fire support, search and rescue, area survey, and electronic warfare. Sensor buoys and decoys are expendable assets. Fuel is finite — manage your flight deck carefully.

Controls

  • Tactical map (canvas): Shows ship, drone positions with type-specific icons, sensor coverage circles, waypoint routes, sensor buoys, and decoy positions.
  • Drone cards: One card per airborne drone showing callsign, type, status, fuel/hull/ammo/cargo bars, mission info, and action buttons.
  • Hangar section: Shows drones on deck with turnaround progress (refuel/rearm/repair) and Launch or Rush Launch buttons.
  • Launch Drone: Click LAUNCH on a hangar drone card.
  • Recall Drone: Click RECALL to order the drone back for recovery and turnaround.
  • Set Waypoint: Click the map with a drone selected to set its destination. Use W key for waypoint mode, P key for patrol route.
  • Deploy Decoy: Click DEPLOY DECOY, then click the map to set the deployment direction. Decoys expire after ~30 seconds.
  • Deploy Buoy: Survey drones can deploy persistent sensor buoys at their current position (B key).

Keyboard shortcuts: 1-9 select drone, L launch, R recall, W waypoint mode, P patrol route, D deploy decoy, B deploy buoy, E toggle engagement rules, C clear to land, Tab cycle drones, Esc cancel.

Tips: Plan patrol routes that maximise coverage within fuel constraints. Recall drones before they reach bingo fuel. Rush turnaround to get drones airborne faster, but they launch with partial fuel/ammo. Coordinate with Science for sensor coverage and Weapons for fire support targets.

ELECTRONIC WARFARE

Role summary: You degrade enemy combat effectiveness through ECM jamming and network intrusion. Jamming reduces enemy firing accuracy over time; network intrusion can temporarily disable an enemy's systems. You also deploy countermeasures to defeat incoming torpedoes.

Controls

  • ECM tactical map (canvas): Ship at centre, enemy contacts as triangles. Dashed ring shows current effective jam range.
  • Jam target selection: Click an enemy on the map or in the enemy list to designate a jam target. Jam factor builds over time while the target is in range.
  • Countermeasures toggle: Enable / disable active countermeasures. Each activation depletes a charge (maximum 10).
  • Intrusion target: Select a jam-target and a target system (propulsion, weapons, etc.), then click Begin Intrusion to launch the Network Intrusion puzzle.
  • Network Intrusion puzzle: Triggered by Begin Intrusion. Successful completion stuns the target for multiple seconds.

Keyboard shortcuts: F1 — help overlay.

Tips: Jam factor maxes out at 80% — beyond that, switch focus to a second enemy. Countermeasure charges don't regenerate; save them for confirmed torpedo volleys. ECM Suite must have adequate power from Engineering or jam range shrinks significantly.

TACTICAL OFFICER

Role summary: You are the ship's information officer. You synthesise data from all stations into tactical assessments and relay them to the crew. You do not directly control any system — your value is situational awareness and coordination. A skilled Tactical officer prevents mistakes more than they create successes.

Controls

  • Tactical display (canvas): North-up sector map showing all contacts, vectors, and hazards. Identical in scope to the Captain's map.
  • Threat priority panel: Ranked list of enemy contacts by estimated threat level (based on range, heading, and type).
  • Objective tracker: Mission objective status in real time — Pending, Active, Completed, or Failed.
  • Relay panel: Send text annotations to other stations via the cross-station notification system.

Keyboard shortcuts: F1 — help overlay.

Tips: Call out shield frequency to Weapons as soon as Science completes a scan. Monitor the objective tracker and warn the Captain if a time-limited objective is about to expire. Your station draws the same sensor data as Captain — if they're busy managing crew, you're their second set of eyes.

HAZARD CONTROL

Role summary: You manage hull damage mitigation. When the ship takes hits, rooms catch fire, decompress, or suffer structural damage. You dispatch repair teams (DCTs) to repair them before the damage spreads. Fire spreads on its own if ignored; decompressed rooms are unrecoverable.

Controls

  • Room grid: Displays all 20 rooms (5 decks × 4 rooms), colour-coded by status: Normal (dim), Damaged (amber), Fire (red), Decompressed (dark).
  • Dispatch DCT: Select a damaged or burning room and click Dispatch. The DCT begins an 8-second repair cycle. Only one DCT per room.
  • Cancel DCT: Recalls the team from the selected room, freeing them for a higher-priority location.
  • Active DCTs panel: Shows all active teams with names, room names, and repair progress bars (0–100%).

Keyboard shortcuts: F1 — help overlay.

Tips: Prioritise fire over damage — fire spreads every 20 seconds to adjacent rooms and cannot be reversed once a room decompresses. A decompressed room cannot be repaired; seal its connecting doors (via Security) to prevent further hull loss. Keep an eye on all decks, not just the one currently taking hits.

5. Environment

SECTOR GRID & NAVIGATION

The game world is divided into a named sector grid. Each sector covers a rectangular region of world space. The Captain's map and Tactical display show the full sector grid with six fog-of-war visibility states:

  • Unknown: Sector has never been observed.
  • Transponder: A friendly beacon has auto-revealed basic info about this sector.
  • Scanned: Science has completed a long-range scan that covers this sector.
  • Surveyed: Science has completed a sector sweep while adjacent to this sector.
  • Active: Your ship is currently inside this sector.
  • Visited: Your ship has previously been in this sector.

Navigation plots a route between sectors. The route is displayed on the Helm minimap as waypoints. Sectors with hazards generate route warnings; Science can perform a sector sweep to reveal the exact hazard layout before Helm commits to a path.

Crew coordination: Science → long-range scan to reveal adjacent sectors before a transit. Helm plans the approach. Captain monitors the strategic map. Flight Ops can launch drones ahead into unknown sectors as a forward sensor picket.

SPACE STATIONS & DOCKING

Multiple station types can appear in the game world, each with different services, hull ratings, and faction alignments:

  • Military: High hull. Restricted services; may require identification.
  • Civilian / Trade Hub: Open to all. Standard commercial services.
  • Research: Specialist science data and sensor packages.
  • Repair Dock: Full hull repair and system services. Primary resupply point.
  • Derelict: No active services; may contain salvage. Requires Science BIO scan to confirm safety before docking.
  • Enemy: Hostile. Cannot dock; must be neutralised or captured by Security boarding.

Docking services run in parallel while docked. Available services include hull repair (45 s), system repair (15 s per system), torpedo resupply (30 s), medical transfer (45 s), atmospheric resupply (30 s), drone service (40 s), EW database update (30 s), crew rest (60 s), sensor data package (30 s), and intel briefing (30 s). Not all stations offer all services — check the station listing before docking.

Enemy station assault: Enemy stations have defensive components — shield generators, turrets, torpedo launchers, and a garrison. The typical assault sequence: EW jams the sensor array, Weapons destroys shield generators, Security boards and captures the station.

SPACE CREATURES

Five creature types roam the game world. Each requires a different response from the crew:

6. Missions

Missions are the core game unit — each mission gives the crew a set of objectives to complete before a defeat condition triggers. Objectives can be sequential, parallel, or conditional, and some missions branch based on crew decisions or performance.

Mission types (overview)

  • Combat missions: Destroy enemy waves, defend friendly stations, or escort assets through contested space. Weapons, Engineering, and EW carry the load; Science provides targeting data; Medical handles casualties.
  • Station assault missions: Attack and capture enemy installations. Typical sequence: EW jams sensors → Weapons destroys shield generators → Security boards and captures. Station may call reinforcements if not jammed in time.
  • Exploration missions: Locate signals, scan anomalies, and navigate hazard fields. Science, Helm, and Flight Operations lead; Hazard Control manages the inevitable scrapes. May require visiting multiple grid sectors or triangulating a signal from two positions.
  • Creature encounter missions: Interact with space creature populations — study void whales, negotiate or sedate rift stalkers, clear hull leeches from a derelict, or disperse a swarm infestation. Science and Comms take the primary roles; Weapons is last resort.
  • Medical missions: Respond to outbreaks, perform triage, and resupply emergency stations. Medical and Science are primary; Comms negotiates access.
  • Diplomatic missions: Negotiate with factions through Communications, fulfil trust-building objectives across multiple stations, and avoid armed conflict. All stations contribute.

Branching. Some missions have conditional tracks that activate or deactivate based on crew actions. For example, a mission might offer a peaceful resolution if Comms hails a faction within a time limit, or escalate to combat if they don't. Pay attention to mission text — the objectives tell you what options are available.

Defeat conditions. Most missions end in defeat when hull reaches zero, a time limit expires, or a key objective fails (e.g. a station you were defending is destroyed). Check the objective panel on the Tactical Officer's station for the active defeat condition.

Training missions. A suite of 12 single-role training missions walks new players through each station individually. The server runs automated AI crew for all other roles so you can focus on learning yours. Access them from the lobby by selecting a training mission from the mission list.

Save & resume. The host can save the current game state from the admin dashboard or lobby. Saved games appear in the lobby's RESUME SAVED GAME section (host-only). Select a save and click RESUME to restore the full game state including ship position, hull, crew, mission progress, and sector grid visibility.

POST-MISSION DEBRIEF

After a mission ends (victory or defeat), each station displays a VIEW DEBRIEF button. Clicking it opens the debrief page in a new tab with a full mission analysis:

  • Result badge: VICTORY or DEFEAT, with mission duration, final hull percentage, and difficulty level.
  • Awards: Per-station awards earned during the mission (e.g. "Sharpshooter" for Weapons, "Lifesaver" for Medical). Awards are computed from the game log and highlight exceptional station performance.
  • Key moments: A timestamped timeline of major mission events — first contact, hull breaches, objective completions, crew deaths, and turning points.
  • Captain's log: All log entries the Captain typed during the mission, presented in chronological order.
  • Station stats table: Per-station action counts showing total actions and top event type for each role.
  • Captain's Replay: An animated minimap canvas that plays back the ship's path through the mission. Use the scrubber to jump to any point in time. Key moments appear as gold markers along the route. Playback speed is adjustable.
Tips: The debrief data is stored in your browser's localStorage. It persists until you play another mission. Share your screen during the replay to review tactical decisions as a crew — it's the best way to improve for next time.

7. Puzzles

Puzzle overlays appear on specific stations when triggered by mission events. They pause interaction with that station's normal controls until resolved. Puzzles have a time limit; if it expires, the puzzle fails — which may fail the associated objective. Some puzzles can receive assist data from other stations.

PuzzleStationMechanic
Circuit Routing Engineering Route power through a grid of conduits by enabling/disabling cells to create a continuous path from source to destination. BFS validation confirms connectivity.
Frequency Matching Science Tune amplitude and frequency sliders for each waveform component until the composite signal matches the target. RMS difference must fall below a tolerance threshold. Comms can relay component data as an assist.
Transmission Decoding Communications A set of cipher symbols map to unique digits 1–9. Sum equations provide constraints. Deduce all mappings. Successfully decoded transmissions relay signal data to Science's Frequency Matching puzzle.
Triage Medical Multiple patients present symptoms. Identify the correct pathogen for each patient from symptom patterns, then prescribe the correct treatment steps in order. Science can assist by revealing pathogen data.
Route Calculation Helm A grid of cells is partially hidden. Some cells contain hazards. Plot a safe path from start to end by clicking cells in order. Hidden cells are revealed on click; a hazard terminates the path attempt. Science can assist by revealing specific hidden cells.
Tactical Positioning Security Place marine squads in rooms before a boarding event. A simulation runs internally to score the placement against intruder paths. Science can reveal optimal interception points.
Firing Solution Weapons A moving enemy contact appears on a radar canvas. Rotate a firing line to the predicted intercept bearing, accounting for target velocity and heading. Science can assist by revealing velocity data, widening the tolerance arc.
Cross-station assists: Science is the central assist station. When a puzzle starts on another station, Science may receive a notification to send assist data. The receiving station sees an "assist available" banner. Assists widen tolerances, reveal hidden information, or pre-fill answers — always worth using.

8. Admin Guide

The admin dashboard at /admin/ provides game management controls accessible from any browser without claiming a crew role. It is intended for teachers, facilitators, and session organisers.

Pause / Resume. The PAUSE button freezes the game loop and sends a PAUSED overlay to all connected stations. All timers stop. Use this for breaks, teaching moments, or to prevent a defeat while troubleshooting. Resume restores normal play instantly.

Difficulty presets. Four presets are available: Cadet, Officer, Commander, and Admiral. Each preset adjusts over 25 gameplay parameters including enemy damage, repair speed, resource levels, scan times, event pacing, and more. Difficulty is normally set in the lobby before launch but can be changed mid-game from the dashboard — changes take effect immediately and all connected stations receive a notification. See the full Difficulty Levels reference for a complete parameter breakdown.

Save & Resume. Click SAVE to snapshot the current game state to disk. Saves are stored in the saves/ directory. To resume, use the Saves section of the admin dashboard or POST to /saves/resume/{save_id}. Only one game can be running at a time.

Player profiles. Each player's callsign is automatically tracked. After each mission, their profile updates with games played, wins, losses, and any new achievements unlocked. View profiles at /profiles or export all profiles as CSV from /profiles/export.

Annotations. Send a text message to a specific station role from the admin dashboard. The message appears as a toast notification on that station's screen. Useful for giving hints or scenario instructions without breaking immersion.

Engagement monitoring. The dashboard shows the last interaction time for each station, the tick count, and a snapshot of ship systems. Use this to identify stations that may have lost connection or are inactive.

Mission editor. The visual mission editor at /editor/ allows creating and editing missions without writing JSON. See the editor's built-in help for full documentation. Missions are saved to the missions/ directory and available in the lobby immediately.
