What is Starbridge?

Starbridge is a cooperative multiplayer bridge simulation game. Players take on specialised crew roles aboard a starship — Captain, Helm, Engineering, Weapons, Science, Medical, Security, Communications, Flight Operations, Electronic Warfare, Operations, and Hazard Control — and must work together to complete missions.

Each crew member operates only their own station. You cannot see what other stations look like; you can only see what your role provides. Information must be communicated verbally between players in real time. The game is designed for co-located crews (around a table) as well as remote play over voice chat.

Starbridge is built for educational and training contexts. Missions can be authored in a visual graph editor, difficulty is tunable, and a full post-mission debrief shows per-station statistics, key moments, and Captain's log entries. Training missions walk new players through individual stations with automated AI crew filling other roles.

Version History

v0.01 Core gameplay loop. Seven original stations (Captain, Helm, Engineering, Weapons, Science, plus lobby and viewscreen), basic enemy AI, combat system, first mission set, and WebSocket infrastructure. 331 tests.
v0.02 Crew health model, ship interior, Medical station, puzzle engine (7 puzzle types), Security & boarding, Communications, the Diplomatic Summit mission, disease outbreak system, ship hazards, EMP & nuclear torpedoes, Captain's log and nuclear authorisation. 948 tests.
v0.03 Polish and expansion. Flight Operations, Electronic Warfare, Operations, Hazard Control stations added. Training missions (12). Debrief dashboard and Captain's Replay. Ship class framework (7 classes). Multi-role support and role-switching. Mission briefing room. 1,578 tests.
v0.04 Tools and infrastructure. Mission Graph Engine (parallel, branch, conditional node types). Visual mission editor with graph canvas, trigger builder, and validation. Save & resume system. Player profiles and achievements. Admin dashboard with pause/resume, difficulty control, and engagement monitoring. Accessibility pass (colour-blind mode, reduced motion, ARIA labels). Landing page and documentation. 1,988 tests.
v0.05 Space, exploration, and creature systems. Sector grid with six-level fog-of-war and sector-aware navigation. Science sector scan (30 s sweep / 90 s long-range; EM, GRAV, BIO, SUB modes). Seven space station types with full docking state machine (clearance, sequencing, docked, 10 services). Five environmental hazard types (nebula, minefield, gravity well, radiation zone, asteroid field). Eight torpedo types — homing, ion, piercing, heavy, proximity, nuclear, experimental — plus point defence (9th ship system). Enemy stations with shield arcs, turrets, and boarding interior. Five space creature types (void whale, rift stalker, hull leech, swarm, leviathan) with unique AI and interaction paths (study, sedate, EW disperse, communicate). Station assault missions, creature encounter missions, sandbox creature spawning, and four new story missions. Balance pass. 2,863 tests.
v0.06 Shield and Medical overhaul. 2D shield focus control — four-facing shields (fore, aft, port, starboard) with drag-to-focus canvas UI; shield distribution shifts dynamically based on focus point. Contact classification on weapons radar (hostile/unknown/friendly/neutral with colour coding). Medical station complete rewrite — individual named crew members with body-region injuries, five treatment types (first aid, stabilise, surgery, intensive care, quarantine), body diagram with severity-colour wireframe, injury degradation and death timers, triage prioritisation, quarantine isolation slots, crew roster system, and full keyboard-driven workflow. Full difficulty system (28 tunable parameters across 4 presets). Crew factor tied to system effectiveness. Comms station overhaul with signal queue, progressive decoding, diplomacy channels, and dynamic mission generation from decoded transmissions. Flight Operations overhaul with 5 drone types, drone AI, patrol routes, buoys, decoys, and cross-station integration. Janitor secret station. 4,821 tests.
v0.07 Per-class ship stats and loadout. Seven ship classes with distinct hull, armour, power, weapon, shield, sensor, and engine profiles. Unique ship systems (stealth, ECM modules, flag bridge, spinal weapons, hospital bay, flight control). Pre-mission loadout screen for equipment selection. Per-ship-class interior layouts (7 JSON floor plans). Consumable resource system (7 resource types: fuel, torpedoes, medical supplies, provisions, repair materials, drone parts, EW charges). Vendor system with 8 vendor types and 6-modifier pricing. Negotiation system with counter-offers, bluff, and barter. Salvage system (wrecks, assessment, extraction, booby traps). Resource allocation and rationing (4 levels). 5,939 tests.
v0.08 Ship environment and cross-station integration. Per-room atmosphere system (O², pressure, temperature, contamination). Radiation system (3 sources, 4 zone tiers, decontamination). Structural integrity per section (5 severity states, collapse cascade, reinforcement). Emergency systems (bulkheads, emergency power, life pods). Hazard Control station overhaul with three-panel layout and 6 overlay modes. Full cross-station integration — priority target markers, general orders, boarding alerts, fire suppression power gates, breach repair coordination, and intelligence pipeline. Mission environmental actions (8 types: fire, breach, radiation, structural damage, contamination, system damage, crew casualty, transmission). Mission editor completeness (15 new trigger types, 5 new entity types, 8 new action types). Signal audit and 46 integration signal-flow tests. 6,831 tests.

Tech Stack

Backend language Python 3.12
Web framework FastAPI + Starlette + uvicorn
Data validation Pydantic v2
Real-time comms WebSocket (game loop), HTTP REST (editor, admin, profiles)
Frontend Vanilla JavaScript (ES modules) — no framework, no build step
Graphics HTML5 Canvas 2D API
Fonts Share Tech Mono (Google Fonts)
Test framework pytest + pytest-asyncio + FastAPI TestClient (httpx)
Persistence JSON files (missions, saves, profiles) — no external database
Logging JSONL event logs per session, post-mission debrief analysis
